# PVDasm x64 (64-bit) Opcode Coverage

This document lists all instruction sets and opcodes supported by PVDasm when disassembling 64-bit PE executables. It builds upon the full x86 coverage documented in [x86.md](x86.md). Every entry below has been verified against the source code.

---

## Inherited from x86

All instruction sets documented in [x86.md](x86.md) are supported in 64-bit mode, including:

- Full 1-byte opcode map (with 64-bit adjustments noted below)
- x87 FPU (0xD8–0xDF)
- 2-byte opcodes (0x0F xx) including MONITOR, MWAIT, FEMMS, RSM, LSS/LFS/LGS
- 3DNow!
- MMX, SSE, SSE2, SSE3, SSSE3, SSE4.1, SSE4.2
- VEX / AVX / AVX2 / FMA3
- EVEX / AVX-512 (partial)
- K-register opmask operations

---

## REX Prefix (0x40–0x4F)

Fully supported. The REX prefix is decoded to access extended registers and 64-bit operand sizes.

| Bit | Name | Effect |
|-----|------|--------|
| Bit 3 (0x08) | REX.W | 64-bit operand size |
| Bit 2 (0x04) | REX.R | Extension of ModR/M reg field (accesses R8–R15, XMM8–XMM15, etc.) |
| Bit 1 (0x02) | REX.X | Extension of SIB index field |
| Bit 0 (0x01) | REX.B | Extension of ModR/M r/m, SIB base, or opcode reg field |

**Note:** In 64-bit mode, opcodes 0x40–0x4F are REX prefixes instead of INC/DEC reg. The INC/DEC functionality is still available via 0xFE/0xFF (Group 4/5) with ModR/M encoding.

---

## Extended Registers

### General-Purpose Registers
| Register | Encoding | Notes |
|----------|----------|-------|
| R8–R15 | REX.B=1 | 64-bit extended GPRs |
| R8D–R15D | REX.B=1 (no REX.W) | 32-bit forms |
| R8W–R15W | REX.B=1 + 66h prefix | 16-bit forms |
| R8B–R15B | REX.B=1 | 8-bit low forms |
| SPL, BPL, SIL, DIL | REX prefix (any) | Uniform byte registers (replace AH/CH/DH/BH when REX present) |

### SIMD Registers
| Register | Encoding | Notes |
|----------|----------|-------|
| XMM8–XMM15 | REX.R/B/X or VEX | Via VEX or REX prefix |
| YMM8–YMM15 | VEX.R/B/X | 256-bit AVX |
| ZMM0–ZMM31 | EVEX.R/R'/B/X | 512-bit AVX-512 |

---

## RIP-Relative Addressing

Fully supported. In 64-bit mode, ModR/M encoding with mod=00, r/m=101 (5) produces RIP-relative addressing instead of the disp32 direct address used in 32-bit mode.

- PVDasm correctly resolves RIP-relative offsets to absolute addresses
- Displayed as `[RIP+disp32]` with the resolved target address

---

## 64-bit Specific Instructions

| Opcode | Instruction | Notes |
|--------|------------|-------|
| 0x63 | MOVSXD r64,r/m32 | Sign-extend 32-bit to 64-bit (replaces ARPL in 64-bit mode) |
| 0F 01 F8 | SWAPGS | Swap GS base with KernelGSBase MSR (ring 0 only) |
| 0F 05 | SYSCALL | 64-bit fast system call |
| 0F 07 | SYSRET | Return from SYSCALL |
| REX.W + 0F C7 /1 | CMPXCHG16B m128 | 16-byte compare-and-exchange (REX.W distinguishes from CMPXCHG8B) |

## 64-bit Operand Size Variants

The following instructions have 64-bit forms in long mode:

| Instruction | 32-bit Form | 64-bit Form |
|------------|-------------|-------------|
| 0x98 | CWDE | CDQE |
| 0x99 | CDQ | CQO |
| 0x9C | PUSHFD | PUSHFQ |
| 0x9D | POPFD | POPFQ |
| 0xA4–A5 | MOVSD | MOVSQ (with REX.W) |
| 0xA6–A7 | CMPSD | CMPSQ (with REX.W) |
| 0xAA–AB | STOSD | STOSQ (with REX.W) |
| 0xAC–AD | LODSD | LODSQ (with REX.W) |
| 0xAE–AF | SCASD | SCASQ (with REX.W) |
| 0xCF | IRETD | IRETQ (with REX.W) |
| 0xE3 | JECXZ | JRCXZ |

## 64-bit Operand Promotions

Certain instructions default to 64-bit operand size in 64-bit mode without requiring REX.W:

| Instruction | Default Size | Notes |
|------------|-------------|-------|
| PUSH r64 (0x50–0x57) | 64-bit | Cannot encode 32-bit PUSH in 64-bit mode |
| POP r64 (0x58–0x5F) | 64-bit | Cannot encode 32-bit POP in 64-bit mode |
| PUSH imm8/imm32 (0x6A/0x68) | 64-bit | Immediate sign-extended to 64-bit |
| CALL rel32 (0xE8) | 64-bit | RIP always 64-bit |
| JMP rel32/rel8 (0xE9/0xEB) | 64-bit | RIP always 64-bit |
| CALL r/m64 (0xFF /2) | 64-bit | Cannot encode 32-bit indirect call |
| JMP r/m64 (0xFF /4) | 64-bit | Cannot encode 32-bit indirect jump |
| PUSH FS/GS (0F A0/A8) | 64-bit | Segment register pushes |
| POP FS/GS (0F A1/A9) | 64-bit | Segment register pops |
| RET (0xC3/0xC2) | 64-bit | RSP always used |
| ENTER/LEAVE (0xC8/0xC9) | 64-bit | RBP/RSP always used |
| Jcc near (0F 80–8F) | 64-bit | RIP always 64-bit |
| LOOP/LOOPcc (0xE0–0xE2) | 64-bit (RCX) | Uses RCX as counter |
| JRCXZ (0xE3) | 64-bit (RCX) | Tests RCX for zero |

| Opcode | 32-bit Instruction | 64-bit Status |

| 0x07 | POP ES | Invalid |
| 0x0E | PUSH CS | Invalid |
| 0x16 | PUSH SS | Invalid |
| 0x17 | POP SS | Invalid |
| 0x1E | PUSH DS | Invalid |
| 0x1F | POP DS | Invalid |
| 0x27 | DAA | Invalid |
| 0x2F | DAS | Invalid |
| 0x37 | AAA | Invalid |
| 0x3F | AAS | Invalid |
| 0x60 | PUSHAD | Invalid |
| 0x61 | POPAD | Invalid |
| 0x62 | BOUND | Invalid |
| 0xD4 | AAM | Invalid |
| 0xD5 | AAD | Invalid |
| 0xD6 | SALC | Invalid |

## Known 64-bit Gaps

All previously listed 64-bit specific gaps have been resolved:
- **SWAPGS** — now decoded (moved to "64-bit Specific Instructions" above)
- **CMPXCHG16B** — now distinguished from CMPXCHG8B when REX.W is present
- **INTO (0xCE)** — verified already correctly flagged invalid (added to invalid instructions table above)

---

## Gaps Shared with x86 (Both Modes)

See the "Known Gaps" section of [x86.md](x86.md) for the complete list of remaining unimplemented extensions:

- TSX extended (XBEGIN, XABORT)
- CET extended (RDSSPD/Q, INCSSPD/Q)
- AVX-512 extended subsets (DQ, BW full, VL, IFMA, VBMI)
- GFNI, VAES/VPCLMULQDQ
